# AI Discovery Feature — Design Spec

> **Date**: 2026-03-23
> **Status**: Draft
> **Author**: Steve + Claude

## Overview

Add AI-powered video discovery to Channel Zero using the user's own Claude API key. Two new interaction modes — **Channel Dial** and **VHS Tape** — let Claude suggest YouTube clips based on watch history, verified in real-time, and streamed to the client via SSE.

## Goals

- Let users discover new content beyond the curated 349-scene library
- Keep the experience thematic — AI features are "pirate signal" hardware on the CRT TV
- BYOK (bring your own key) model — no server-side API keys, no cost to the app owner
- Ephemeral playback, persistent tapes — dial clips are live, tapes build a shelf

## Non-Goals

- Replacing or contaminating the existing Blast Me engine
- Building a full YouTube search integration (no YouTube API key required)
- Server-side storage or user accounts
- Next.js migration (stays Vite + React)

## Architecture

### Infrastructure Change

Migrate from GitHub Pages to Vercel. The Vite SPA deploys identically (auto-detected framework, `base: "/"`). Vercel provides serverless functions for the AI pipeline.

### File Structure Additions

```
api/
├── dial.js          # Streaming POST endpoint — channel dial quick discovery
├── tape.js          # Streaming POST endpoint — VHS tape curated session
├── validate.js      # Lightweight key validation endpoint (POST, non-streaming)
└── _lib/
    ├── claude.js    # Claude API client (receives key per-request)
    ├── verify.js    # YouTube oEmbed verification helper
    └── prompts.js   # Prompt templates — imports FILTERS from shared data for valid vocabulary
```

### Streaming Transport

The browser's `EventSource` API only supports GET requests and cannot set custom headers. Since we need to POST a request body (watch history, filters) and send the API key via `Authorization` header, we use **`fetch()` with `ReadableStream`** instead.

The Vercel function responds with `Content-Type: text/event-stream` and writes SSE-formatted chunks. The client reads the response body as a stream, parsing `data: {...}\n\n` lines manually. This gives us POST + custom headers + streaming — all three.

```js
// Client-side streaming pattern (simplified)
const res = await fetch("/api/dial", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": `Bearer ${apiKey}`
  },
  body: JSON.stringify({ watchHistory, currentVibes })
});
const reader = res.body.getReader();
const decoder = new TextDecoder();
// Read chunks, parse SSE "data:" lines, dispatch scenes
```

### Data Flow

```
┌─────────────┐   POST (fetch)     ┌──────────────────┐
│   Browser    │ ──────────────────▷ │  Vercel Function  │
│              │  Authorization hdr  │                   │
│  - History   │                     │  1. Build prompt  │
│  - Filters   │                     │  2. Call Claude    │
│              │  ReadableStream     │  3. Parse JSON    │
│              │ ◁────────────────── │  4. Verify each   │
│  ScenePlayer │  (SSE format)       │  5. Stream valid  │
└─────────────┘                     └──────────────────┘
                                           │
                                     ┌─────┴─────┐
                                     │ Claude API │
                                     │ (user key) │
                                     └───────────┘
```

## UI Design

### Channel Dial (Right Side of TV)

A rotary knob on the right side of the TV bezel.

- **Visual**: Metallic/bakelite knob with numbered position indicators. Past the numbers is a red/glitchy "pirate zone" marked with interference symbols.
- **Interaction**: Click the pirate zone (or drag the knob past the numbers). Screen fills with heavy static. As Claude finds valid clips, the static briefly clears and a clip plays for its full duration. Spin again to skip to the next signal. Clips auto-advance as they stream in.
- **Feel**: Like tuning a radio through static. Signals fade in and out.

### VHS Slot (Below the TV Screen)

A horizontal slit below the screen with a visible tape edge peeking out.

- **Visual**: Tape has a hand-written label generated by Claude (e.g., "LATE NIGHT CHAOS MIX", "CERTIFIED UNHINGED VOL. 3").
- **Interaction**: Click the tape to push it in. Screen shows a blue VCR tracking screen with rolling horizontal lines, then the first clip starts. Clips play in sequence (4-6 clips). A VCR-style counter ticks in the corner.
- **Feel**: Found footage from a garage sale mystery tape.

### Tape Shelf

- Tapes save to localStorage when the stream completes (`[DONE]` received and at least 1 verified clip). If the user ejects mid-stream or the stream returns zero valid clips, the tape is not saved.
- Max 5 tapes (oldest evicted on overflow).
- Small cassette icons render near the VHS slot. Click any to replay without API cost.
- Each cassette shows its generated name on hover.

### Mode Switching

When either AI feature is active, the "Blast Me" button dims and shows "SCANNING..." (dial) or "PLAYING TAPE..." (tape). Clicking "Blast Me" exits AI mode and returns to normal.

### Unpowered State

Before an API key is connected, the dial and VHS slot are always visible but appear "unpowered" — dim, no glow. Once a key is connected, they gain a subtle glow/hum indicating the hardware is live.

## API Key Entry — "The Service Panel"

A themed key entry experience that fits the CRT aesthetic.

### Interaction

1. Small screw/latch icon on the bottom-right corner of the TV bezel.
2. Clicking it "flips" the TV with a CSS 3D rotation, revealing the **back panel**.
3. Back panel shows:
   - A sticker: `MODEL: CH-4NN3L-Z3R0` with a fake serial number
   - A port labeled `SIGNAL DECODER` with an input field styled as a hardware port
   - A status LED: red (no key), yellow (validating), green (connected)
4. User pastes their Claude API key into the port.
5. On paste, a lightweight validation call fires (tiny prompt to verify the key works).
6. LED goes green, panel auto-flips back after 1 second.

### First-Time UX

Clicking the dial or tape without a key → TV flips to back panel. Port pulses with label: `"CONNECT DECODER TO RECEIVE PIRATE SIGNALS"`.

### Storage & Security

- Key saved in localStorage (`sisb-api-key`).
- Sent to Vercel functions via `Authorization` header (HTTPS only).
- Vercel function passes it directly to Anthropic API — never logged, stored, or inspected.
- No key ever appears in URLs, query parameters, or server logs.

## AI Pipeline

### `/api/dial.js` — Channel Dial

1. **Input**: `{ apiKey, watchHistory: [...last 20 scene IDs], currentVibes: [...active filters] }`
2. **Prompt**: Claude receives the user's recent patterns (titles, vibes, eras resolved from scene IDs) and active filters. Instructed to suggest 5-8 YouTube clips: videoId, start/end timestamps (15-45 sec clips), quote, description, vibes, era, source. Emphasis on variety — different vibes and eras than recent watches.
3. **Verification**: Each suggestion checked via YouTube oEmbed (`https://www.youtube.com/oembed?url=https://www.youtube.com/watch?v={videoId}&format=json`). Returns 200 if video exists and is embeddable; 404/401 otherwise. Video ID format validated first (11 chars, `[A-Za-z0-9_-]`).
4. **Streaming**: Each verified clip sent as SSE event: `data: {"scene": {...}}\n\n`. Failed verifications silently dropped. Stream ends with `data: [DONE]\n\n`.

### `/api/tape.js` — VHS Tape

1. **Input**: `{ apiKey, watchHistory: [...last 50 scene IDs], favorites: [...favorited scene IDs] }`
2. **Prompt**: Deeper analysis. Claude receives more history plus favorites. Instructed to identify a taste profile, then generate a themed tape: a name, theme description, and 4-6 clips that follow a mood arc.
3. **First SSE event**: Tape metadata: `data: {"tape": {"name": "...", "theme": "..."}}\n\n`
4. **Subsequent events**: Verified clips, same as dial.
5. **Stream end**: `data: [DONE]\n\n`

### `/api/validate.js` — Key Validation

1. **Input**: POST, non-streaming. API key via `Authorization: Bearer` header (same as dial/tape — no key in request body).
2. Sends a minimal prompt to Claude (`"Respond with OK"`) using the provided key
3. Returns `{ valid: true }` on success, `{ valid: false, error: "..." }` on failure
4. Used by the Service Panel on key paste

### Prompt Design

The Vercel functions resolve scene IDs to full scene objects before prompting Claude. This gives Claude rich context (titles, descriptions, vibes, eras) rather than opaque IDs. The client sends the full curated scene objects for the relevant IDs (not just IDs) so the server doesn't need a copy of `scenes.js`.

`prompts.js` includes the complete list of valid vibes and eras (imported from a shared vocabulary constant) in every prompt, instructing Claude to only use values from that list. This prevents Claude from inventing vibe/era keys that don't exist in `filters.js`.

The prompts instruct Claude to return a JSON array. Scene IDs from Claude responses are prefixed with `ai-dial-{timestamp_ms}-{index}` or `ai-tape-{timestamp_ms}-{index}` (using millisecond timestamps to avoid collisions).

### Model Choice

Use **Claude Haiku 4.5** (`claude-haiku-4-5-20251001`) for both endpoints. Reasons:
- Speed: ~1-2s response vs 3-8s for Sonnet — critical for staying within Vercel function timeouts
- Cost: ~10x cheaper than Sonnet — friendlier for BYOK users
- Quality: Haiku 4.5 is more than capable of suggesting well-known YouTube videos and timestamps
- Timeout safety: Vercel Hobby plan has a 10s function timeout. Haiku + oEmbed checks fits comfortably; Sonnet may not.

The model ID is specified in `prompts.js` so it can be changed in one place if needed.

### Cost

- Dial call: ~1-2K tokens, ~$0.002
- Tape call: ~2-4K tokens, ~$0.005-0.01
- Verification: oEmbed is free, ~50ms per check

## Data Model

### AI Scene Object

Same schema as curated scenes, plus metadata fields:

```js
{
  id: "ai-dial-1711234567-0",
  videoId: "dQw4w9WgXcQ",
  start: 42,
  end: 71,
  quote: "Never gonna give you up",
  description: "The ultimate internet rickroll moment",
  vibes: ["pure-nostalgia", "chaotic-good"],
  era: "viral-classics",
  source: { title: "Rick Astley - Never Gonna Give You Up", year: 2009 },
  _ai: true,
  _verified: true,
  _discoveredAt: 1711234567000
}
```

### State Management

**React state (ephemeral — controls playback):**

| Key | Type | Purpose |
|-----|------|---------|
| `aiMode` | `null \| "dial" \| "tape"` | Current AI mode |
| `dialResults` | `Scene[]` | Clips from current dial session |
| `dialIndex` | `number` | Current clip in dial rotation |
| `tapeData` | `{ name, theme, clips[] }` | Current tape being played |
| `tapeIndex` | `number` | Current clip in tape |

**localStorage (persistent):**

| Key | Schema | Cap | Purpose |
|-----|--------|-----|---------|
| `sisb-api-key` | `string` | 1 | Claude API key |
| `sisb-tapes` | `Tape[]` | 5 | Saved tape shelf |
| `sisb-ai-discoveries` | `Scene[]` | 50 | Rolling log of all AI-discovered clips (for dedup + favoriting) |

### Integration with Existing Systems

**History isolation**: AI clips do NOT feed into `useBlastEngine` history or `useWatchHistory`. They are a separate track. The watch history sent to the AI endpoints comes from the curated-only `useWatchHistory` (resolved to full scene objects client-side before sending). This means after a long AI session, the taste signal sent to Claude still reflects the user's curated viewing patterns, not a feedback loop of AI suggestions.

**Favorites**: AI clips CAN be favorited. The `useFavorites` hook continues storing IDs only (no schema change). Resolution happens in `App.jsx` before passing to `FavoritesList`: merge the curated `SCENES` array with the current `sisb-ai-discoveries` log into a single lookup array, then pass that merged array as the `scenes` prop. `FavoritesList` itself needs no code changes — it already does `.find()` on its `scenes` prop. If an AI clip has been evicted from the 50-cap discoveries log but is still favorited, the `.find()` returns nothing and the favorite shows a "signal lost" placeholder (a small UI addition to `FavoritesList`). This is an accepted trade-off — the feature is ephemeral-first, and 50 entries is generous for typical usage.

**ScenePlayer**: Receives AI scenes with zero prop changes — same schema. The `onBlast` callback behavior changes based on `aiMode`:
- `aiMode === null`: Normal behavior — calls `getNext()` on the blast engine
- `aiMode === "dial"`: Advances `dialIndex`. If at the end of `dialResults`, exits AI mode (sets `aiMode` to `null`)
- `aiMode === "tape"`: Advances `tapeIndex`. If at the end of the tape's clips, exits AI mode

This routing happens in `App.jsx` where the `onBlast` handler already lives — it checks `aiMode` and dispatches accordingly. No new props on `ScenePlayer`.

**Service Panel DOM**: The CSS 3D flip wraps the entire TV at the `App.jsx` level (a container div around `ScenePlayer` + controls), not inside `ScenePlayer` itself. This avoids restructuring `ScenePlayer`'s internal DOM.

### Dial Clip Lifecycle

When the user activates the dial:
1. Screen shows static. `fetch()` fires to `/api/dial`.
2. As verified clips stream in, they append to `dialResults`.
3. The first clip starts playing immediately when it arrives.
4. When the current clip ends (`onBlast` fires), advance to the next clip in `dialResults`.
5. If the next clip hasn't arrived yet (buffer empty), show static until it streams in.
6. If the stream is complete (`[DONE]` received) and no more clips remain, exit AI mode.
7. Spinning the dial again during playback: cancel the current stream, clear `dialResults`, start a new request.

### localStorage Keys (Complete)

**New keys (this feature):**

| Key | Schema | Cap | Purpose |
|-----|--------|-----|---------|
| `sisb-api-key` | `string` | 1 | Claude API key |
| `sisb-tapes` | `Tape[]` | 5 | Saved tape shelf |
| `sisb-ai-discoveries` | `Scene[]` | 50 | Rolling log of AI clips (dedup + favorite resolution) |

**Existing keys (unchanged):**

| Key | Schema | Cap | Purpose |
|-----|--------|-----|---------|
| `sisb-blast-history` | `string[]` | 200 | Blast engine scoring history |
| `sisb-favorites` | `string[]` | — | Favorited scene IDs (unchanged — AI IDs resolved via `sisb-ai-discoveries`) |
| `sisb-watch-history` | `object[]` | 50 | Watch history with timestamps |

## Error Handling

All errors are themed to the CRT TV aesthetic.

| Scenario | TV Behavior |
|----------|-------------|
| Invalid API key | Back panel LED blinks red. Label: `"NO SIGNAL — CHECK DECODER"` |
| Claude API rate limit / error | Rolling static + text: `"SIGNAL LOST — TRY AGAIN"`. Auto-clears after 3s |
| All clips fail verification | Dial: `"DEAD AIR — SPIN AGAIN"`. Tape: `"BAD TAPE — EJECT AND TRY AGAIN"` |
| Some clips fail verification | Silently dropped. Fewer results, no error shown |
| Network offline | Fullscreen static: `"NO SIGNAL"` |
| Video dies after discovery | Existing error handling (codes 100/101/150) auto-advances. Last clip → exit AI mode |
